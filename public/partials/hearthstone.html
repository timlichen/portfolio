Hearthstone E-Commerce made with LAMP stack.

This E-Commerce web app was created with a team at Coding Dojo at the end of a 3 week PHP course. This project gave our team an opportunity to showcase what we've learned and further explore API, jQuery integration and development workflows.

We combined the PHP MVC Framework CodeIgniter with Bootstrap, mySQL, jQuery, and a Mashape Hearthstone card game API.

Prioritizing

We analyzed a provided wireframe and prioritized key features. Our team only had a few days to complete this project and we wanted to determine which features were must haves and which features would be nice to haves if we had additional time.

Next, we choose a product to "sell". We knew we wanted to integrate an API into our project. This decision drove our usage of the Mashape Hearthstone card game API. The API provided us with lots of card images and data to work with so we could focus on building the application rather than generating generic placeholder text and images.

Database Planning

Consideration went into how we wanted to manage the shopping cart quantities, how it would be updated and how we wanted to keep track of our customers. Our biggest decision at this point was to decide how we wanted to use the card data received from the API. After some deliberation and trial and error we decided upon using session data for customer identification, cart inventory and quantities only writing to the database when an order was placed. In regards to the card data we chose to use a small subset of the available cards mainly because there were some discrepancies with the card data we got back from the API and since the data wasn’t constantly changing we didn’t want the performance of the application to be too dependant on the API.
